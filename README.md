## Navigating the Files

**Literature and References** -- All journal articles, books, and reference PDFs used in the literature review are stored in the `reference/` folder.

**Interview Data** -- Qualitative data in the form of interview transcripts are in the `research files/` folder. These files contain sensitive participant information and should be handled in accordance with UQ ethical research guidelines.

**Survey Data and Analysis** -- Quantitative materials including raw survey data (CSV files), the survey questions document, and the Python analysis script are in the `STUFF/` folder.

**Figures and Visualisations** -- Charts, screenshots, and figures generated during the research are in `new folder/`.

**Drafts and Reports** -- The final submitted paper (`paper submitted.pdf`), the final Word version (`FINAL PAPER V9999.docx`), and the final proposal (`proposal ABOSOLUTE FINAL.docx`) are in the root directory. An earlier draft proposal is in `not sure arrange later/`.

**Project Logbook** -- `Project_logbook.txt` in the root directory is used to log team contributions, decisions, and version notes.

## How to Contribute

### Getting Started

1. **Fork this repository** by clicking the "Fork" button at the top right of this GitHub page. This creates your own copy to work in.
2. **Clone your fork** to your local machine using GitHub Desktop or the terminal:
   ```
   git clone https://github.com/YOUR-USERNAME/Genertive_AI_Research_49595916.git
   ```
3. **Create a new branch** before making any changes. Use a descriptive name that reflects what you are working on:
   ```
   git checkout -b your-descriptive-branch-name
   ```

### Making Changes

- Make your edits to the relevant files in your local branch.
- Write a clear, descriptive commit message explaining what you changed and why. For example:
  - `Added cleaned survey response data for Q3`
  - `Updated interview transcript for participant Jason`
  - `Fixed data type errors in analysis script`
- Commit and push your changes to your fork:
  ```
  git add .
  git commit -m "Your descriptive message here"
  git push origin your-descriptive-branch-name
  ```

### Submitting a Pull Request

1. Go to the original repository on GitHub.
2. Click **"New pull request"** and select your branch.
3. Write a brief description of what your changes include and why.
4. The repository owner will review your pull request, and may request changes or approve and merge it.

### Resolving Conflicts

If your pull request has a conflict with the main branch, the reviewer will flag this. Work with the reviewer to resolve conflicts by selecting the appropriate version of the content, then commit and push the resolution.

### File Naming Conventions

Please follow these conventions when adding new files to keep the repository organised:

- Use lowercase letters and underscores instead of spaces (e.g., `survey_data_wave2.csv`)
- Include a date or version number where relevant (e.g., `analysis_script_v2.py`, `interview_transcript_2024-08-20.docx`)
- Avoid vague names like "final", "new", or "updated" without additional context
- Place files in the correct folder according to the structure above

### Sensitive Data

Interview transcripts and consent forms contain participant information. Do not share these files publicly or add them to a public repository without appropriate de-identification and ethics approval. If you are unsure, contact the project lead before uploading.
